## 三、快速开始（3 步）

### 步骤 1：初始化数据库

确保 MySQL 已启动，root 密码为 `123456`（如果不是，先修改 `application.yml` 和下方命令中的密码）。

```bash
# 一条命令完成建库 + 建表 + 数据 + 存储过程/触发器/视图/函数
mysql -u root -p123456 --default-character-set=utf8mb4 < init-all.sql
```

如需自定义密码，修改 `library-server/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/library_db?...
    username: root
    password: 你的密码          # ← 改这里
```

### 步骤 2：启动后端

```bash
cd library-server
mvn spring-boot:run
```

看到 `Started LibraryApplication` 即表示成功，后端运行在 **http://localhost:9090**。

### 步骤 3：启动前端

```bash
cd library-web
npm install          # 首次运行需要安装依赖
npm run dev
```

看到 `VITE v5.x ready` 即表示成功，前端运行在 **http://localhost:3000**。

> **一键启动**：Windows 下直接双击 `start.bat`，会自动依次启动后端和前端，等待后端就绪后自动启动前端。
> 
> **一键停止**：双击 `stop.bat` 可同时停止后端和前端进程。
> 
> | 脚本          | 功能                                                          |
> | ----------- | ----------------------------------------------------------- |
> | `start.bat` | 启动后端（mvn spring-boot:run）→ 等待 9090 端口就绪 → 启动前端（npm run dev） |
> | `stop.bat`  | 查找端口 9090/3000 的进程 PID → taskkill 强制终止                      |

## 九、常见问题

### Q1: 启动后端报 "Access denied for user"

> 检查 `application.yml` 中 MySQL 用户名密码是否正确。

### Q2: 端口被占用

> ```bash
> # Windows 查看端口占用
> netstat -ano | findstr ":9090"
> netstat -ano | findstr ":3000"
> # 或双击 stop.bat 一键停止
> ```

### Q3: 前端报 "Cannot find module 'vite'"

> 未安装依赖，在 `library-web` 目录下执行 `npm install`。

### Q4: 数据库字符集乱码

> MySQL 连接必须指定 `--default-character-set=utf8mb4`。

### Q5: 换电脑如何迁移？

> 1. 安装 JDK 17 + Maven + Node.js + MySQL
> 2. 执行 `mysql < init-all.sql` 初始化数据库
> 3. `mvn spring-boot:run` 启动后端
> 4. `npm install && npm run dev` 启动前端

### Q6: 如何打包部署？

> ```bash
> # 后端打包
> cd library-server && mvn package -DskipTests
> java -jar target/library-server-1.0-SNAPSHOT.jar
> 
> # 前端打包
> cd library-web && npm run build
> # 将 dist/ 目录部署到 Nginx
> ```
